# 🤖 CareerBot — План разработки

## Результаты исследования

### HH.ru API — возможности и ограничения

**❌ Создать резюме с нуля через API — НЕЛЬЗЯ**
HH.ru API не имеет endpoint для создания нового резюме программно. Это ограничение платформы.

**✅ Что МОЖНО через API (с OAuth-авторизацией соискателя):**
- `GET /resumes/mine` — список резюме пользователя
- `PUT /resumes/{id}` — редактировать существующее резюме (title, skills, experience)
- `POST /resumes/{id}/publish` — обновить/поднять резюме
- `GET /vacancies` — поиск вакансий (анонимно, без авторизации)
- `GET /vacancies/{id}` — полная информация о вакансии
- `POST /negotiations` — откликнуться на вакансию с выбранным резюме
- `GET /employers/{id}` — информация о компании

**Варианты обхода для "автосоздания":**
1. Пользователь создаёт 1 пустое резюме на hh.ru → бот заполняет через PUT (рекомендую)
2. Selenium/Playwright автоматизация (хрупко, могут забанить)
3. Генерировать PDF/DOCX офлайн + пользователь загружает руками

**Рекомендация:** Гибридный подход — бот генерирует адаптированное резюме в 2 форматах:
- 📄 **1-страничный PDF** (красивый, для прямой отправки)
- 📋 **Текст для HH** (готовый к копипасте или PUT через API если OAuth подключён)

### Стек и библиотеки
- **python-telegram-bot** v21+ (async) — Telegram бот
- **Anthropic Claude API** — AI-движок для анализа и генерации
- **SQLite + SQLAlchemy** — БД профилей пользователей
- **reportlab / weasyprint** — генерация PDF
- **python-docx** — генерация DOCX
- **httpx** — HTTP-клиент для HH API / web research
- **BeautifulSoup** — парсинг вакансий если нужен fallback

## Архитектура

```
Telegram Bot (BOT_ID)
    ↓
Command Router
    ├── /start → Онбординг (сбор контекста)
    ├── Загрузка документов → Парсер (PDF/DOCX/TXT)
    ├── "Готово" → AI создаёт цифровой профиль
    ├── Ссылка/текст вакансии → Анализатор вакансий
    └── "Готовь резюме" → Генератор резюме
    
SQLite DB
    ├── users (id, tg_id, name, created_at)
    ├── profiles (user_id, full_profile_json, summary, skills, experience...)
    ├── documents (user_id, type, content, parsed_at)
    ├── vacancies (user_id, url, company, analysis_json, created_at)
    └── resumes (user_id, vacancy_id, format, file_path, created_at)
```

## Пользовательский флоу

### Фаза 1: Онбординг (первый запуск)

```
Пользователь: /start
Бот: Привет! Я помогу с поиском работы. Для начала мне нужно узнать тебя.

📎 Загрузи документы (PDF, DOCX, TXT):
- Резюме (любое, даже старое)
- Рабочие тетради курсов
- Должностные инструкции
- Сертификаты, дипломы

Можешь загрузить несколько файлов, я обработаю все.
```

```
Бот: Теперь расскажи текстом:
1. Чем нравится заниматься на работе?
2. Чем НЕ нравится?
3. Хобби и интересы?
4. Семья, возраст? (опционально)
5. Какую зарплату хочешь?
6. Готов ли к переезду/релокации?
7. Формат работы: офис/удалёнка/гибрид?
```

```
Пользователь: [отвечает, загружает файлы]
Пользователь: Готово ✅

Бот: Анализирую... (30-60 сек)

Бот: 📊 Твой цифровой профиль готов!

[Краткая сводка: ключевые компетенции, целевые позиции, 
сильные стороны, зоны роста, рекомендованный диапазон зарплат]

Всё верно? Хочешь что-то изменить?
```

## Мультипользовательский режим

### Как добавить нового пользователя:
1. Александр добавляет tg_id в конфиг-файл `allowed_users.json`
2. Пользователь пишет /start → проходит онбординг
3. У каждого свой профиль в БД, изолированный контекст

```json
// allowed_users.json
{
  "users": [
    {"tg_id": 292168972, "name": "Александр", "role": "admin"},
    {"tg_id": 123456789, "name": "Анна", "role": "user"}
  ]
}
```

## Вопросы для обсуждения

### 1. OAuth для HH.ru — нужен?
- **Без OAuth:** бот парсит вакансии по URL, генерирует файлы офлайн
- **С OAuth:** бот может обновлять резюме на HH, откликаться на вакансии, видеть историю
- OAuth требует регистрацию приложения на dev.hh.ru
- **Рекомендация:** начать БЕЗ OAuth, добавить потом

### 2. AI-движок
- Claude API (Anthropic) — лучше для русского, глубокий анализ
- OpenAI — альтернатива
- **Рекомендация:** Claude (уже есть OAuth, знаем API)

### 3. Шаблон 1-страничного резюме
- Жду образец от тебя
- Пока сделаю минималистичный шаблон (имя, контакты, summary, опыт, навыки)

### 4. Деплой
- Systemd сервис на текущем VPS
- Или Docker
- **Рекомендация:** systemd, как upload2server бот

### 5. Материалы с Google Drive
- Жду ссылку, обработаю как часть онбординга
